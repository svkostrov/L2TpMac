# L2TP Office — L2TP-клиент для macOS (без IPsec)

Нативное macOS-приложение (SwiftUI, окно + значок в menu bar) для подключения к чистому L2TP-серверу **без IPsec** — например, к офисному туннелю, аналогичному интерфейсу `L2TP0` на Keenetic. Внутри используется `pppd` для PPP-интерфейса и встроенный user-space L2TP helper; системные VPN-профили macOS и глобальный `/etc/ppp/options` не меняются.

<img src="L2TPOfficeApp/AppIcon_1024.png" width="96" alt="Иконка L2TP Office">

## Зачем, если в macOS есть VPN

Штатный клиент macOS умеет только **L2TP over IPsec** и требует shared secret. Если на сервере IPsec нет (`security-level public` на Keenetic, чистый L2TP на Cisco ISR и т.п.), системным интерфейсом подключиться нельзя. Это приложение решает именно такой случай.

## Возможности

- Подключение/отключение в один клик из окна или из menu bar (замок в строке меню: закрыт — туннель активен).
- Рабочий режим — **split-туннель**: в туннель заворачиваются только указанные CIDR-сети (по умолчанию `172.16.99.0/24`), остальной трафик и DNS идут напрямую.
- Настройки подключения переживают перезапуск приложения.
- Пароль VPN можно временно показать/скрыть в поле ввода.
- Автозапуск при входе в систему и автоподключение при старте приложения.
- Живой лог PPP/L2TP прямо в окне (`/tmp/l2tp-office-app.log`, pppd с `debug` + сообщения helper).
- Права root — через ограниченный root-helper. При первом подключении приложение один раз попросит пароль администратора, установит helper в `/Library/PrivilegedHelperTools` и sudoers-правило только для него; последующие подключения/отключения идут без повторного системного пароля.
- Защита от чужих PPP-туннелей: приложение управляет только своими pppd (маркер в argv) и не убивает сторонние.
- Восстановление сети после отключения: очистка стейл-записей сетевого стора, восстановление default route и DNS (см. BR-18 в [bugreports.md](bugreports.md)).
- При выходе из приложения активный туннель корректно отключается перед завершением.

## Установка

1. Скачай `L2TP-Office-<версия>.zip` из [Releases](https://github.com/svkostrov/L2TpMac/releases), распакуй, перенеси `L2TP Office.app` в `/Applications`.
2. Подпись ad-hoc, поэтому при первом запуске: ПКМ по приложению → «Открыть» → «Открыть» (или `xattr -cr "/Applications/L2TP Office.app"`).
3. macOS 13+; приложение собирается universal для Apple Silicon и Intel.

При первом запуске укажи сервер, логин, пароль и сети для split-туннеля, затем нажми «Подключить» — появится системный запрос пароля администратора.

## Сборка из исходников

```bash
git clone https://github.com/svkostrov/L2TpMac.git
cd L2TpMac
./build-install.command   # компиляция, сборка бандла, подпись, установка в /Applications
```

Скрипт собирает `L2TPOfficeApp/main.swift` и `L2TPOfficeHelper/main.go`, кладёт иконку и Info.plist, подписывает ad-hoc **после** копирования ресурсов (иначе ломается подпись и Dock показывает белый квадрат), устанавливает в `/Applications` и сбрасывает кэш иконок.

## CLI-скрипты (без GUI)

В репозитории есть и консольный вариант:

```bash
sudo ./l2tp-connect.sh      # подключить
sudo ./l2tp-disconnect.sh   # отключить
ifconfig ppp0               # статус
```

Лог: `/tmp/l2tp-office.log`. Пароль в скрипте хранится открытым текстом — правь под себя и не выкладывай.

## Технические заметки

**User-space L2TP backend.** Начиная с версии 1.13 приложение не использует Apple `L2TP.ppp`: `pppd` запускается поверх `pty`, а отдельный helper внутри `.app` сам устанавливает UDP/L2TPv2-сессию и мостит PPP-кадры. Это сделано, чтобы не трогать системный L2TP over IPsec и обойти проблему, где ICMP через Apple L2TP-плагин работал, а TCP зависал в `SYN_SENT`.

**Соответствие опций Keenetic → pppd:**

| Keenetic | pppd |
|---|---|
| `peer <ip>` | `remoteaddress` |
| `authentication identity/password` | `user` / `password` |
| `lcp echo 30 3` | `lcp-echo-interval 30`, `lcp-echo-failure 3` |
| `no ccp` | `noccp` |
| `ipcp default-route` | `defaultroute` |
| `ipcp dns-routes` | `usepeerdns` |
| `ip mtu 1400` | `mtu 1200`, `mru 1200` |

**Восстановление сети (BR-18).** Если pppd умирает, не откатив закоммиченный сетевой стор, configd продолжает считать мёртвый PPP-сервис основным: default route деградирует до interface-scoped, глобальный DNS пустеет — интернет пропадает. Скрипты подключения и отключения чистят стейл-ключи `State:/Network/Service/*/IPv4`, восстанавливают default route (с повторами) и DNS через обновление DHCP-аренды. Подробности — в [bugreports.md](bugreports.md).

## Известные ограничения

- В split-режиме DNS-имена офисной сети не резолвятся (DNS сервера не используется) — обращайся по IP.

- **1.49** — лог PPP/L2TP сохраняется от запуска приложения до закрытия и больше не очищается при переподключениях.
- **1.38** — стабильный транспорт L2TP через текущий системный путь к интернету, включая активные внешние VPN-туннели; добавлено автопереподключение при обрыве.
- **1.13** — внедрён user-space L2TPv2 helper: `pppd` работает через `pty`, без Apple `L2TP.ppp` и без изменения системных VPN-профилей macOS.
